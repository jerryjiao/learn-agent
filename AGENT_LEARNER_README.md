## 🚀 开始学习

### 基础命令

```bash
# 开始学习 (从第一个课程或当前位置继续)
/learn

# 学习指定课程
/learn 01-1    # MCP工具集成
/learn 01-2    # 从零构建智能体框架
/learn 02-1    # LangGraph基础
...

# 查看所有可用课程
/status
```

### 测试和复习

```bash
# 测试当前课程 (默认中等难度)
/quiz

# 分级测试
/quiz easy     # 3题基础概念
/quiz medium   # 7题 (基础+理解)
/quiz hard     # 10题 (全面测试)

# 复习已学课程
/review 01-1
```

### 进度管理

```bash
# 查看详细进度报告
/status
```

## 🎯 学习流程

### 1. 开始学习
```
用户: /learn 01-1

Claude:
  📚 开始学习: 01-1 MCP工具集成

  【核心概念1: 什么是MCP?】
  ...讲解...

  ❓ 问题1: MCP的全称是什么?
  用户: Model Context Protocol
  Claude: ✅ 正确!

  【核心概念2: 通信模式】
  ...讲解...

  ...继续...

  【综合测试】(10题)
  ...

  🎉 得分: 85/100

  ✅ 笔记已生成: notes/01-mcp-tool.md
```

### 2. 生成笔记
每次学习完成后自动生成,包含:
- 核心概念总结
- 关键要点
- 代码示例
- 测试题与答案
- 相关资源链接

### 3. 追踪进度
进度自动保存到 `data/progress.json`:
```json
{
  "current_project": "01-1",
  "progress": {
    "01-1": {
      "status": "completed",
      "quiz_score": 85,
      "started_at": "2026-01-13T10:00:00"
    }
  }
}
```

学习后的笔记示例 (保存在 `notes/01-mcp-tool.md`):

学习日期: 2026-01-13
学习状态: ✅ 已完成
测试得分: 85/100

## 📚 核心概念

### 1. MCP协议
- **定义**: Model Context Protocol (模型上下文协议)
- **作用**: 标准化AI与外部工具的交互
- **优势**: 统一接口、易于扩展

### 2. 通信模式
- **stdio**: 本地执行,低延迟
- **sse**: 网络访问,多用户

## 💡 关键要点
1. MCP是"万能翻译器"
2. 两种通信模式适用不同场景
3. 项目结构分客户端和服务端

## ❓ 自测题

答案: Model Context Protocol
...

## 🔄 学习建议

### 初次学习
1. 按顺序学习 (`/learn 01-1` → `01-2` → `02-1`...)
2. 每个课程学习完成后做测试 (`/quiz medium`)
3. 查看生成的笔记,加深理解

### 复习巩固
1. 使用 `/review [课程ID]` 复习已学课程
2. 重点复习错题部分
3. 定期查看 `/status` 了解整体进度

### 进阶学习
1. 完成基础课程后尝试 `/quiz hard`
2. 结合实际项目练习
3. 阅读教案中的完整代码

## 🛠️ 技术细节

### 数据结构

**课程索引** (`curriculum/index.json`):
- 5个模块
- 12个项目
- 包含难度、主题、代码路径等信息

**进度文件** (`data/progress.json`):
- 当前学习位置
- 每个项目的完成状态
- 测试成绩和学习时长

### 笔记模板

使用 `templates/note.md` 作为模板,支持:
- 动态变量填充
- 代码高亮
- 结构化内容
